## 什么是 Cowork

Cowork 是 Claude Desktop 的 agentic 功能模式（研究预览）。与标准 Chat 模式不同，Cowork 可以：

- **自主执行多步骤任务**：分析请求、拆分子任务、并行处理、交付结果
- **直接读写本地文件**：无需上传下载
- **生成专业文档**：Excel（带公式）、PowerPoint、格式化报告等
- **执行长时间运行的任务**：无超时限制
- **定时执行重复任务**：通过 scheduled tasks

### Cowork vs Chat

| 特性 | Chat | Cowork |
|------|------|--------|
| 适用场景 | 头脑风暴、快速问答 | 需要实际完成的工作 |
| 文件访问 | 需要上传 | 直接访问本地文件 |
| 执行能力 | 有限 | 完整（运行代码、生成文件） |
| Token 消耗 | 较少 | 较多（计算密集） |
| 定时任务 | 不支持 | 支持 |

**简单说：Chat 是你的思考伙伴，Cowork 是你的执行伙伴。**

## 系统要求与设置

### 平台支持

- **Claude Desktop**：macOS 和 Windows x64
- **不支持**：Windows ARM64、Linux、纯 Web
- **需要**：付费计划（Pro、Max、Team、Enterprise）

### 开始使用

1. 打开 Claude Desktop
2. 找到模式切换器（Chat / Cowork 标签页）
3. 点击 Cowork 进入任务模式
4. 描述你的任务
5. 审查 Claude 的执行方案后开始

## 核心功能

### 本地文件操作

Cowork 在一个轻量级 Linux VM 中运行，可以安全地执行代码，同时通过 workspace folder 访问你选择的本地文件夹。

**授权文件访问：**
- 选择一个本地文件夹给 Cowork 访问
- Claude 可以读取和写入该文件夹中的文件
- 输出文件会保存到你选择的文件夹

### 专业文档生成

Cowork 可以创建：
- **Excel 文件**（.xlsx）：带公式、格式化、数据分析
- **PowerPoint 演示文稿**（.pptx）：带布局、图表、样式
- **Word 文档**（.docx）：带目录、页眉页脚、样式
- **PDF 文件**
- **HTML/React 可视化**
- **各种代码文件**

### 定时任务

```
/schedule
```

创建重复执行的任务。在侧栏的「Scheduled」部分查看和管理。

**注意：** 定时任务仅在电脑保持唤醒状态且 Claude Desktop 处于打开状态时运行。

### Sub-agents 并行处理

Cowork 可以将工作拆分给多个 sub-agents 并行处理。实测中，10 个文件的并行处理可以将 ~30 分钟的等待缩短到 ~4 分钟。

### 项目组织

使用 Projects 功能组织相关工作，获得持久的上下文和记忆。

### 移动端访问

Pro/Max 用户可以通过 Claude iOS 应用远程分配任务给 Cowork。

## 配置与自定义

### 全局指令

Settings → Cowork → Edit（Global instructions）

设置适用于所有会话的固定指令，如：
- 输出语调偏好
- 默认输出格式
- 角色上下文

### 文件夹指令

选择本地文件夹时可以添加项目特定的上下文说明，Claude 可以在会话中更新这些指令。

### 插件

通过插件自定义 Cowork 的行为：
- 打包 skills、connectors、sub-agents
- 针对特定角色或团队的定制

### 连接器

Cowork 可以通过连接器接入外部应用：
- Gmail
- Notion
- Google Calendar
- Figma
- Slack
- 等等

## 使用技巧

### 1. 从简单开始，逐步扩展

先从简单的文件夹集成开始，然后：
1. 逐步添加连接器
2. 根据需要构建自定义 skills
3. 设置定时任务自动化重复工作

### 2. 给出足够上下文

Cowork 对任务的理解取决于你的描述质量：

**不好：**
```
整理我的文件
```

**好：**
```
扫描我的 Downloads 文件夹，按文件类型分类整理：
- PDF 放到 Documents/PDF/
- 图片放到 Pictures/
- 超过 30 天未修改的文件移到 Archive/
- 列出移动了哪些文件
```

### 3. 批量处理以节省 Token

Cowork 消耗的 token 较多，建议：
- 将相关工作合并到单个会话
- 对复杂多步骤任务使用 Cowork
- 快速问答用 Chat 模式

### 4. 善用审查步骤

在执行前：
- 审查 Claude 的执行方案
- 敏感文件操作要特别注意
- Claude 在永久删除文件前会请求确认

### 5. 常见使用场景

**文件管理：**
```
整理文件夹、处理收据、批量重命名文件
```

**研究与分析：**
```
综合信息、分析访谈记录、提取模式
```

**数据处理：**
```
统计分析、数据可视化、数据转换、异常检测
```

**文档创建：**
```
生成带公式的电子表格、演示文稿、格式化报告
```

## 常见问题与限制

### 技术限制

| 限制 | 说明 |
|------|------|
| 记忆范围 | 仅在 Projects 内持久化，独立会话不跨会话保存 |
| 共享 | 不能与他人共享会话 |
| 运行要求 | 必须桌面应用保持打开，关闭 = 任务停止 |
| 合规性 | 不适用于受监管工作负载（无审计日志和数据导出） |
| 对话历史 | 本地存储，不在标准数据保留时间范围内 |

### 常见疑问

**"Setting up Claude's workspace" 加载很慢？**
→ 正常的版本更新过程，等待即可。

**任务突然停止了？**
→ 检查应用是否关闭或系统是否进入睡眠。

**Token 用量消耗很快？**
→ 将 Cowork 保留给真正复杂的任务。简单问答用 Chat。

**找不到输出文件？**
→ 检查文件访问权限和你指定的输出位置。

### 安全注意事项

Cowork 运行在虚拟机中，提供了与主系统的隔离。但：
- Claude 确实可以访问你授权的本地文件
- 审查所有操作后再批准，特别是涉及敏感文件时
- 作为研究预览，agent 技术仍在发展中
