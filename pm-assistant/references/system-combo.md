# 世游核心系统 - 系统架构

> 来源：世游核心系统 - 系统架构.docx

## 架构概述

Combo 于 2023-12-29 上线，包含以下核心服务：

| 服务 | 职责 |
|-----|------|
| **api 服务** | API Gateway，所有对外 REST API 的统一入口 |
| **central 服务** | 整个核心系统的中心服务，存储管理所有核心业务配置 |
| **auth 服务** | 处理账号与登录等所有用户身份相关的业务逻辑 |
| **order 服务** | 处理订单与发货等所有充值相关的业务逻辑 |
| **payment 服务** | 与第三方支付平台交互，处理世游收款的支付流程 |
| **console 服务** | 世游发行平台 Web 管理控制台，调用所有后端服务的 gRPC 接口 |

## SQS 消息队列

引入 Amazon SQS 实现以下目标：

- 利用 SQS 的 visibility timeout 实现订单发货的通知重试，重试间隔随重试次数增大
- 将 order 服务和 payment 服务解耦
- 缓冲消息，以恒定速度处理支付通知与发货重试

Queue 类型选择 FIFO queues。

**权限方面**：通过服务级别的 AWS IAM Role 精细控制服务对每个 FIFO queue 的 API 调用权限。

**性能方面**：
- 消费 FIFO queue 中的消息时，总是使用 long polling
- order 服务支持配置每个 FIFO queue 的 consumer 数量（即 long polling 的 goroutine 数量）
