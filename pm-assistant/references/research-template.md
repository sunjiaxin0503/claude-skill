# 调研分析文档参考结构

这份文档是调研分析报告的参考结构，实际输出时应根据需求特点和与用户的沟通来灵活调整。

---

## 中度调研参考结构

适用于需求目标明确、对 1-2 款竞品进行功能拆解的场景。

```markdown
# [需求名称] 调研分析报告

## 1. 调研背景

[一句话说明需求背景和调研目的]

## 2. 竞品功能拆解

以竞品的核心功能为目录来组织，每个功能说清楚「怎么做的」和「为什么这么做」。

### 2.1 [核心功能A]

#### 竞品X 的做法
- **功能描述**：[具体怎么实现的，用户操作流程]
- **设计意图**：[为什么这么做，解决了什么问题]

#### 竞品Y 的做法
- **功能描述**：[具体怎么实现的]
- **设计意图**：[为什么这么做]

#### 对比小结
[两者在这个功能上的核心差异，各自的优劣，对我们的启发]

### 2.2 [核心功能B]
[同上结构]

### 2.3 [核心功能C]
[同上结构]

## 3. 调研结论

### 3.1 关键发现
[3-5 条最重要的调研发现]

### 3.2 设计建议
[基于调研结果，对我们的方案设计提出具体建议]

### 3.3 风险提示
[调研中发现的潜在风险或需要注意的问题]

## 4. 参考资料
[列出所有参考的链接、文档、资料来源]
```

---

## 重度调研参考结构

适用于从 0 开始做新系统、或需要了解全新领域的场景。在中度调研的基础上增加行业与技术调研。

```markdown
# [需求名称] 调研分析报告

## 1. 调研背景

[一句话说明需求背景和调研目的]

## 2. 竞品功能拆解

[同中度调研，以核心功能为目录组织]

## 3. 行业与技术调研

### 3.1 行业趋势
[相关领域的发展趋势、行业标准、最佳实践]

### 3.2 技术方案参考
[相关的技术实现方案、开源项目、技术文章总结]

### 3.3 用户反馈与需求验证
[来自用户社区、应用商店评论、论坛讨论等的用户真实反馈]

## 4. 调研结论

### 4.1 关键发现
[3-5 条最重要的调研发现]

### 4.2 设计建议
[基于调研结果，对我们的方案设计提出具体建议]

### 4.3 风险提示
[调研中发现的潜在风险或需要注意的问题]

## 5. 参考资料
[列出所有参考的链接、文档、资料来源]
```
