# 世游用户运营 Gamer - 系统架构

> 来源：世游用户运营 Gamer 系统架构.docx

## 架构概述

Gamer 是世游的用户运营系统，提供面向玩家的运营活动、俱乐部、社区等功能。

## 核心服务

### api 服务
新的网关服务，没有持久化数据，如有需要则使用 Redis 缓存。主要职责：

- **提供 REST API**：组合 Gamer 下的服务和其他系统的 gRPC 方法，供前端网页和小程序调用
- **Token 验证与签发**：根据前端在世游通行证 web-api 获取的 ID Token，验证用户身份后签发 Gamer Token
- **跨游戏项目通用 API**：Gamer Token 中包含 GameId，实现多游戏项目通用
  - 通用路径：/v1/..
  - 活动路径：/v1/event/:event/..
  - 俱乐部路径：/v1/club/..
  - 社区路径：/v1/community/..
- **跨域处理**：内置 allowlist，仅允许 allowlist 中的 Origin 跨域

### event 服务
运营活动服务，主要职责：
- 按运营需求完成活动逻辑
- 记录活动产生的用户参与、任务、奖励发放等数据
- 提供 hook 地址，接收第三方平台推送的事件
- 对接 Combo gm 服务，进行游戏内奖励发放

### club 服务
俱乐部服务，主要职责：
- 按游戏项目维护用户基础信息：头像、昵称、个性签名、收货地址
- 实现通用的会员体系和积分系统
- 根据不同游戏项目维护会员和积分计算逻辑、数值
- 实现通用的积分兑换商城
- 对接 Combo gm 服务，进行游戏内奖励发放

### community 服务
社区服务，主要职责：
- 游戏论坛管理
- 用户发帖和互动
- 用户论坛经验等级计算

### weixin 服务
对接微信平台，实现微信相关功能

### data 服务
处理游戏埋点数据

## 系统间关系

- Gamer api 服务依赖**世游通行证** web-api 验证用户身份
- event 和 club 服务对接 **Combo** gm 服务进行游戏内奖励发放
- Gamer Token 中包含 GameId，支持多游戏项目通用
