# 产品需求文档（PRD）模板

以下是 PRD 的标准结构。PRD 的核心价值是让开发团队拿到就能干活，所以每个细节都要足够具体。

## 使用说明

- 每个功能的描述要做到「开发不需要来找产品确认」的程度
- 业务规则中避免使用「等」「相关」「适当」这类模糊词汇，用具体数值和条件替代
- 状态流转务必画出完整的状态机，不能有孤立状态或死循环
- 异常处理要考虑网络超时、并发冲突、数据为空等常见场景
- 对于游戏运营/中台场景，特别注意多游戏/多租户的兼容性
