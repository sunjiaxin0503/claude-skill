# pm-assistant

A Claude skill for guiding product managers through the full requirement delivery process（产品经理助手：从需求收集到文档审查的完整流程）。

---

## 简介

`pm-assistant` 是一个面向产品经理和需求分析人员的 Claude Skill。它通过**对话驱动、逐步深入**的方式，帮助用户将模糊的产品想法一步步转化为可落地的产品需求文档，并在最终阶段以严格的审查视角对文档质量进行把关。

### 适用场景

- 从零开始梳理一个新功能或新系统的需求
- 对已有想法进行调研分析和竞品对比
- 设计技术方案概要，评估可行性
- 编写完整的产品需求文档（PRD）
- 对需求文档进行多维度审查，发现遗漏和风险

### 触发关键词

以下关键词会自动触发本 Skill：

- 产品需求、需求分析、PRD、产品文档
- 方案设计、需求调研、竞品分析
- 需求审查、需求评审、方案概要
- 帮我梳理需求、帮我写 PRD、帮我做调研
- 帮我把这个需求理清楚、帮我出一份需求文档
- 我想做一个 xx 功能、我有一个想法想落地

---

## 功能特性

- **5 阶段完整流程**：需求背景收集 → 调研分析 → 方案概要设计 → PRD 编写 → 需求文档审查
- **对话优先，文档延后**：每个阶段以充分沟通为核心，只有用户明确要求时才生成文档
- **提问驱动**：像顾问一样逐步深入，每次聚焦一个核心问题，避免信息过载
- **灵活的文档结构**：模板仅作参考，实际输出根据具体需求和沟通结果灵活调整
- **自由跳转**：用户可随时说「跳到第 X 步」调整节奏，无需反复确认
- **严格审查**：最终阶段切换为"严厉的技术评审专家"角色，从可实施性、描述清晰度、逻辑完整性、边界情况、多角色视角五个维度进行审查
- **内置系统参考**：包含 Combo、Account、Gamer 等核心系统架构文档，方案设计阶段可直接引用

## 安装方法

### 前提条件

- 已安装 [Claude 桌面版（Cowork）](https://claude.ai/) 或 Claude Code CLI
- 具备导入自定义 Skill 的权限

### 步骤

1. **下载或克隆本仓库**：

   ```bash
   git clone https://github.com/sunjiaxin0503/claude-skill.git
   ```

2. **将 `pm-assistant` 文件夹放入你的 Skills 目录**：

   - Cowork 桌面版：在 Cowork 界面中选择「导入 Skill 文件夹」，选择 `pm-assistant` 目录即可。
   - Claude Code CLI：将文件夹放入 `.claude/skills/` 目录下，或在 Settings 中指定 Skills 路径。

3. **重启 Claude** 使 Skill 生效。

4. **验证安装**：在对话框中输入「帮我梳理一个需求」，如果 Claude 以【第一步：需求背景收集】开始提问，即安装成功。

## 工作流程

### 第一步：需求背景收集

通过提问帮用户把模糊的想法变成清晰的需求描述，涵盖痛点、目标用户、预期效果、现有方案、约束条件和优先级。

### 第二步：调研分析

基于需求背景进行竞品和领域知识调研，支持中度调研（功能拆解）和重度调研（行业趋势 + 技术方案）两种深度。

### 第三步：方案概要设计

结合内置系统架构文档，与用户讨论技术方案的关键决策点：服务选型、数据存储、系统对接等。

### 第四步：产品需求文档（PRD）

将所有沟通成果整合为一份开发团队可直接执行的完整 PRD，逐个模块讨论细节和边界情况。

### 第五步：需求文档审查

以严格的技术评审视角审查 PRD，按🔴严重 / 🟡中等 / 🟢建议三级标注问题，协助修改。
